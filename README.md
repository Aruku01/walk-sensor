# walk-sensor（歩行検知システム）

スマートフォンの加速度センサーで歩行を検知し、その歩数をリアルタイムでPCに送信して音を鳴らすシステムです。ブラウザだけで動作し、アプリのインストールやログインは不要です。

## 特徴

- スマホのブラウザでURLを開くだけで使える（インストール・ログイン不要）
- 加速度センサーで歩行を検知し、着地のタイミングで信号を送信
- Firebase を経由して、離れた場所のPCへリアルタイムに送信
- PC側では受信した歩数をカウントし、足音を鳴らせる

## 構成

このシステムは2つのページで動作します。

| ファイル | 役割 | 開く場所 |
|---|---|---|
| `index.html` | 歩行を検知して送信する | スマートフォン |
| `pc.html` | 受信して音を鳴らす・歩数を表示する | PC |

これらは Firebase Realtime Database を経由して連携します。

## 使い方

### 1. PC側の準備
PCのブラウザで `pc.html` を開きます。音を鳴らす場合は「音を有効にする」ボタンを押します（押さなくても歩数の表示は動きます）。

### 2. スマホ側の準備
スマートフォンのブラウザで `index.html` を開き、「スタート」ボタンを押します。iOSの場合、モーションセンサーへのアクセス許可を求められるので「許可」します。

※ スマホの消音（マナー）モードでは音が鳴らないため、音を使う場合は解除してください。

### 3. 歩く
スマホを持って歩くと、歩行が検知され、PC側の歩数が増えていきます。音を有効にしていれば、着地に合わせて音が鳴ります。

## 歩行検知の仕組み

加速度センサーの値から重力を除いた「動きの加速度」を計算し、着地のたびに現れる山（ピーク）を検出します。単発の動き（持ち上げなど）を歩行と誤検知しないよう、一定のリズムで山が続いたときだけ「歩行中」と判定します。

## 調整項目

`index.html` 内の以下の値で、検知の感度を調整できます。

- `THRESHOLD`：山とみなす動きの大きさ（大きいほど鈍感）
- `REFRACTORY_MS`：1歩の最小間隔（二重カウント防止）
- `RHYTHM_MIN_MS` / `RHYTHM_MAX_MS`：歩行リズムとみなす間隔の範囲
- `NEED_RHYTHM`：歩行と判定するまでに必要な連続回数

## 技術

- HTML / JavaScript（ブラウザのDeviceMotion API）
- Firebase Realtime Database（デバイス間のリアルタイム通信）
- GitHub Pages（HTTPS配信。iOSのセンサー利用にはHTTPSが必須）

## 注意

- iOSでモーションセンサーを使うには、ページがHTTPSで配信されている必要があります（GitHub Pagesはこれを満たします）。
- Firebase の設定情報はブラウザ上で動作させるためのものです。
